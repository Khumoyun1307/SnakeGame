# Security Policy

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead:

1. Use GitHubâ€™s private vulnerability reporting / Security Advisories (if enabled for this repository), **or**
2. Contact the project maintainer(s) privately through a channel listed on their GitHub profile.

When reporting, include:

- A clear description of the issue and potential impact
- Steps to reproduce (or a proof of concept)
- Affected version(s) and your environment (OS, Java version)

## Supported Versions

Security fixes are provided for the latest version on the default branch.
